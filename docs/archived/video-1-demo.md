## Metode Rekam: Screen Studio di PC

Semua rekaman dilakukan di **PC/Mac** menggunakan **Screen Studio** dengan browser dalam **mobile view**. Tidak perlu 2 HP — cukup 2 tab browser.

### Setup Browser

| Tab | Isi | Browser |
|-----|-----|---------|
| **Tab 1 — Owner** | Login ke CatatOrder dashboard | Chrome (normal) |
| **Tab 2 — Pelanggan** | Buka link toko `catatorder.id/slug` | Chrome Incognito (logged out) |

**Mobile view:** DevTools (F12) → Toggle Device Toolbar → pilih iPhone 14 Pro (393x852) atau ukuran serupa.

### Setup Screen Studio

- **Crop area:** Hanya area mobile viewport browser (sembunyikan address bar, DevTools, dan elemen desktop lainnya)
- **Background:** Gunakan background gelap atau blur di Screen Studio supaya terlihat seperti layar HP
- **Cursor:** Sembunyikan cursor mouse, atau gunakan mode "touch dot" di Screen Studio supaya terlihat seperti tap jari
- **Zoom:** Manfaatkan auto-zoom Screen Studio saat klik — ini membuat rekaman terasa lebih hidup
- **Frame rate:** 60fps untuk smooth scrolling

### Tips Screen Studio

- Rekam setiap shot/bagian **terpisah** — lebih mudah di-edit nanti
- Gerakkan mouse **pelan dan natural** seperti jari di HP (jangan gerakan mouse cepat)
- Klik, jangan drag — kecuali untuk demo swipe (tahan klik + geser)
- Pause 0.5 detik setelah setiap aksi supaya penonton bisa lihat hasilnya
- Gunakan **Window capture** (bukan full screen) supaya bisa crop ke mobile viewport saja

## Persiapan

### Yang Dibutuhkan

- PC/Mac dengan **Screen Studio** terinstall
- Browser Chrome dengan DevTools mobile view
- Akun CatatOrder dengan link toko aktif dan 3-4 produk sudah diisi
- WhatsApp Web (web.whatsapp.com) — untuk shot WA (opsional, lihat catatan di bawah)
- Aplikasi edit video (CapCut / DaVinci Resolve) — untuk final edit, text overlay, dan music

### Aset Audio

- Suara notifikasi WhatsApp (ding)
- Suara notifikasi CatatOrder (pesanan masuk)
- Background music: mulai tense/hectic, transisi ke positif/upbeat (cari di CapCut free music)
- Suara efek: tap pop, swipe whoosh, send sound

### Pengaturan Sebelum Rekam

1. Buka CatatOrder di Tab 1 (login) > Pengaturan > pastikan link toko aktif
2. Pastikan sudah ada 3-4 produk di daftar produk
3. Upload QRIS (opsional — tidak ditampilkan di video ini)
4. Buka Tab 2 (incognito) > buka link toko kamu
5. Bersihkan dashboard pesanan (atau biarkan ada beberapa supaya terlihat hidup)
6. Set kedua tab ke mobile view (DevTools > device toolbar > iPhone 14 Pro)
7. Atur Screen Studio: crop ke mobile viewport, sembunyikan cursor atau pakai touch dot

## Catatan tentang Shot WhatsApp

Beberapa shot dalam script melibatkan WhatsApp. Ada 3 opsi:

| Opsi | Cara | Kelebihan | Kekurangan |
|------|------|-----------|------------|
| **A. WhatsApp Web** | Buka web.whatsapp.com di tab terpisah (mobile view) | Terlihat autentik | Perlu setup chat palsu |
| **B. Skip WA shots** | Langsung fokus ke alur CatatOrder, skip bagian WA | Lebih cepat produksi | Kurang konteks "sebelum vs sesudah" |
| **C. Stock footage** | Pakai stock video/template CapCut untuk chat WA chaos | Profesional | Tidak 100% autentik |

**Rekomendasi:** Opsi A untuk opening hook (chat WA berantakan) dan Opsi B untuk shot "kirim ke WA" (cukup tunjukkan preview WA di CatatOrder, cut sebelum WA terbuka — tambah text overlay "Langsung terkirim ke WhatsApp").

## Shot-by-Shot Breakdown

### BAGIAN 1 — Masalah (0-4 detik)

**Tujuan:** Penonton berhenti scroll karena mereka mengenali layar HP mereka sendiri.

---

**[0-1.5 detik] — HOOK**

| | |
|---|---|
| **Visual** | Close-up layar HP. Buka WhatsApp. Chat list dengan banyak pesan belum dibaca. Nama-nama seperti "Kak Dina", "Bu Sari", "Pak Budi". Preview pesan: "Kak pesan nasi box 5 ya", "Mba yang kemarin jadi ga?", "Harganya berapa kak?" |
| **Audio** | Suara notifikasi WhatsApp beruntun: *ding, ding, ding, ding* |
| **Text overlay** | Tidak ada — biarkan visual yang bicara |
| **Cara rekam** | **Opsi A:** Buka WhatsApp Web di tab baru (mobile view). Siapkan beberapa chat dengan pesan pesanan. Screen Studio rekam saat scroll pelan di chat list. **Opsi C:** Skip rekam, pakai stock footage/template CapCut untuk chat WA berantakan. |

> **Psikologi:** Penonton melihat layar yang persis sama dengan HP mereka setiap pagi. Otak langsung bilang "Itu HP saya." Ini membeli 3 detik berikutnya.

---

**[1.5-4 detik] — AMPLIFIKASI MASALAH**

| | |
|---|---|
| **Visual** | Quick cuts (0.5 detik tiap cut): (1) Scroll naik-turun di chat WA cari pesanan. (2) Tulis di buku, coret-coret. (3) Kalkulator HP, ketik angka. (4) Balik ke WA — pesan "Kak udah bayar belum?" centang biru tapi tidak dibalas. |
| **Audio** | Music mulai — beat rendah, terasa hectic/stressful |
| **Text overlay** | "Masih catat pesanan dari chat satu per satu?" |
| **Cara rekam** | (1) WhatsApp Web di mobile view, atau stock footage. (2) Rekam tangan nulis di kertas pakai HP — ini satu-satunya shot yang perlu direkam di luar Screen Studio. (3) Buka app kalkulator web di mobile view. (4) WhatsApp Web atau stock footage. |

> **Psikologi:** Quick cuts menciptakan rasa cemas — menyamai stres harian mereka yang sebenarnya. Text overlay memberikan kata untuk frustrasi yang selama ini tidak mereka sebut.

---

### BAGIAN 2 — Transisi (4-6 detik)

**Tujuan:** Buka rasa penasaran. Penonton HARUS lihat kelanjutannya.

---

**[4-6 detik] — PIVOT**

| | |
|---|---|
| **Visual** | Layar bersih. Transisi: fade to white atau swipe up. Momen tenang sebentar. |
| **Audio** | Music diam 0.5 detik (silence = attention) lalu mulai naik ke nada positif |
| **Text overlay** | "Kalau pelanggan bisa pesan sendiri?" |
| **Cara rekam** | Tidak perlu rekam — buat transisi di CapCut / DaVinci saat editing |

> **Psikologi:** Curiosity gap terbuka. Pertanyaan yang belum dijawab = penonton tidak akan scroll. Mereka butuh tahu "Gimana caranya?"

**[18-19 detik] — NOTIFIKASI DI HP OWNER**

| | |

> **Psikologi:** Momen dopamine. Suara *ding* + visual notifikasi = reward signal. Otak penonton merasakan: "Enak banget kalau pesanan masuk sendiri."

> **Tips rekam:** Cara paling mudah — rekam Tab 1 (halaman Pesanan) terus-menerus. Di jendela lain, submit pesanan dari Tab 2. Toast akan muncul secara realtime di Tab 1. Potong di editing.

**[23-25 detik] — AKSI PESANAN**

| | |

Coba dulu — kalau swipe berfungsi, lebih bagus secara visual. |

> **Psikologi:** Loop tertutup. Penonton sudah lihat perjalanan lengkap: chat berantakan > link > pelanggan pesan > masuk otomatis > konfirmasi siap kirim. Semua dalam 28 detik. Mereka sekarang percaya ini nyata dan mudah.
