# BAGIAN 6: Skenario Prediksi CatatOrder 2026-2028

> Menggunakan framework dari riset "Seni Memprediksi Dunia"

---

## Metodologi

Analisis ini menggunakan kombinasi:
1. **Scenario Planning** (metode Shell/Pierre Wack) — bukan prediksi tunggal, tapi 4 skenario berbeda
2. **Probabilistic Thinking** (Tetlock/Superforecasters) — setiap skenario diberi probabilitas
3. **Constraint Analysis** (Zeihan/Friedman) — eliminasi yang tidak mungkin
4. **Game Theory** (Bueno de Mesquita) — analisis insentif semua aktor
5. **Reference Class Forecasting** (Kahneman) — outside view dari kasus serupa

---

## Skenario 1: BASE CASE — "Tumbuh Pelan tapi Pasti" (45%)

### Narasi
CatatOrder mendapatkan 5-10 user pertama selama Ramadan 2026 melalui outreach manual ke komunitas F&B. Beberapa bertahan aktif setelah Ramadan. Growth lambat tapi steady, didorong oleh word-of-mouth dan konten TikTok. WA viral loop memberikan tambahan moderat. Meta WA Bot belum approved sampai Q3 2026.

### Timeline
```
Mar 2026:  5-10 user pertama (outreach manual + Ramadan)
Jun 2026:  20-40 user aktif
Sep 2026:  50-100 user aktif
Des 2026:  100-200 user aktif
Mar 2027:  200-400 user aktif
```

### Metrics
| Metrik | 6 Bulan | 12 Bulan |
|--------|---------|----------|
| User Aktif | 50-100 | 100-200 |
| MRR | Rp300K-1.4M | Rp1.4M-2.8M |
| Profit/bulan | Rp200K-1.3M | Rp1.3M-2.5M |
| Conversion rate | 3-5% | 5-8% |

### Asumsi Kunci
- Founder konsisten membuat konten TikTok 3-5x/minggu
- WA viral loop menghasilkan 1-2 user baru per 10 user aktif per bulan
- Retention Day-30 di atas 15%
- Tidak ada kompetitor besar yang masuk
- Ekonomi Indonesia stabil (GDP growth 4.5-5.2%)

### Early Warning Signals
- ✅ Menuju skenario ini jika: 5+ user aktif di akhir April 2026
- ⚠️ Diverging jika: <3 user aktif di akhir Mei 2026

## Skenario 2: BULL CASE — "Viral Loop Aktif" (20%)

### Narasi
CatatOrder hit product-market fit kuat di segment katering/bakery. Link toko (catatorder.id/nama-bisnis) mulai tersebar organik karena pelanggan end-user yang menerima struk menjadi curious. Satu konten TikTok tentang "cara kelola pesanan WA" viral (100K+ views), memicu gelombang signup. WA Bot diapprove, enabling otomasi penuh. Sebuah komunitas UMKM besar (HaiBolu/TDA) mengadopsi CatatOrder sebagai rekomendasi resmi.

### Timeline
```
Mar 2026:  10-20 user (Ramadan spike)
Jun 2026:  100-200 user (TikTok + viral loop)
Sep 2026:  500-800 user (community adoption)
Des 2026:  1.000-2.000 user (compound growth)
Mar 2027:  3.000-5.000 user
```

### Metrics
| Metrik | 6 Bulan | 12 Bulan |
|--------|---------|----------|
| User Aktif | 200-500 | 1.000-2.000 |
| MRR | Rp2.8M-8.2M | Rp14M-36M |
| Profit/bulan | Rp2.5M-7.3M | Rp12M-33M |
| Conversion rate | 5-10% | 8-12% |

### Asumsi Kunci
- Satu atau lebih "breakout moment" (TikTok viral, community endorsement)
- WA Bot approved → automasi meningkatkan retention signifikan
- Network effect mulai compound: setiap user membawa 0.3-0.5 user baru/bulan
- Founder mampu handle customer support untuk ratusan user sendirian (atau hire 1 CS part-time)

### Trigger Events yang Dibutuhkan
1. TikTok content viral (>100K views) — dipicu oleh konten yang relatable
2. Endorsement dari komunitas besar (HaiBolu 75K member, TDA 42K member)
3. Meta WA Bot approval
4. Media coverage (tech blog atau berita UMKM)

### Risiko dari Skenario Ini
- Solo founder overwhelmed oleh growth → kualitas support menurun
- Infra perlu upgrade ke Supabase Pro + Vercel Pro (biaya naik ke Rp1.1M/bulan)
- Feature requests membanjir, founder kehilangan fokus

## Skenario 3: SURVIVE CASE — "Hidup tapi Tidak Tumbuh" (25%)

### Narasi
CatatOrder mendapatkan beberapa user, tapi growth stagnasi. Beberapa user churn setelah Ramadan (pesanan turun, kembali ke buku tulis). WA viral loop lemah karena volume pesan terlalu rendah. Founder mulai kehilangan motivasi karena traction lambat. Produk tetap berjalan karena biaya sangat rendah, tapi tidak ada momentum.

### Timeline
```
Mar 2026:  3-5 user (outreach berhasil sebagian)
Jun 2026:  5-15 user (beberapa churn post-Ramadan)
Sep 2026:  10-20 user (growth minimal)
Des 2026:  10-30 user (stagnasi)
```

### Metrics
| Metrik | 6 Bulan | 12 Bulan |
|--------|---------|----------|
| User Aktif | 10-20 | 10-30 |
| MRR | Rp50K-200K | Rp100K-400K |
| Profit/bulan | Break-even atau sedikit positif | Break-even |
| Conversion rate | 1-3% | 2-4% |

### Penyebab Utama
- Target market terlalu resisten terhadap perubahan (status quo bias dominan)
- Konten TikTok tidak berhasil (low engagement, algoritma tidak mendukung)
- WA groups seeding tidak efektif (admin menolak promosi)
- Founder tidak cukup agresif dalam outreach
- Produk ada "hidden friction" yang baru terlihat saat user nyata mencoba

### Early Warning Signals
- ⚠️ Menuju skenario ini jika: Retention Day-7 < 10%
- ⚠️ Menuju skenario ini jika: 0 referral organik dalam 2 bulan pertama
- ⚠️ Menuju skenario ini jika: Founder membuat <2 video TikTok/minggu

### Exit Strategy dari Skenario Ini
1. **Pivot target segment:** Jika F&B tidak berhasil, coba penjahit/konveksi (seasonal demand Lebaran + nikahan lebih predictable)
2. **Pivot channel:** Jika organik tidak berhasil, coba kerjasama dengan PKK/Dharma Wanita/kelompok pengajian (distribusi offline)
3. **Tambah co-founder:** Cari partner yang fokus distribusi sementara founder fokus produk

## Skenario 4: FAIL CASE — "Tidak Ada Traction" (10%)

### Narasi
Upaya distribusi tidak menghasilkan user aktif sama sekali. Beberapa orang sign up tapi tidak ada yang menggunakannya secara rutin. Founder menghabiskan 3-6 bulan tanpa traction, kehilangan motivasi, dan akhirnya meninggalkan proyek atau pivot ke produk lain.

### Penyebab Potensial
1. **Problem-solution fit tidak valid:** UMKM ternyata TIDAK mau pindah dari buku tulis, bahkan yang gratis
2. **Execution gap:** Founder tidak berhasil melakukan outreach yang cukup agresif
3. **Timing salah:** Ramadan 2026 bukan trigger yang cukup kuat
4. **External shock:** Perubahan kebijakan WhatsApp, atau kompetitor besar launch tool gratis
5. **Personal:** Founder burnout, masalah finansial, kehilangan waktu untuk proyek ini

### Probabilitas Rendah (10%) Karena:
- Biaya sangat rendah (Rp62.500/bulan) → tidak ada pressure finansial untuk tutup
- Produk sudah selesai → tidak ada development cost ongoing
- Masalah yang dipecahkan nyata dan tervalidasi oleh riset
- BAHKAN jika gagal, data dan codebase tetap bernilai untuk proyek berikutnya

## Reference Class Forecasting — Outside View

### Kelas Referensi: Micro-SaaS Solo Founder di Emerging Market

Berdasarkan data dari IndieHackers, MicroConf, dan studi micro-SaaS:

| Metrik | Median | Top 25% |
|--------|--------|---------|
| Time to first paying customer | 3-6 bulan | 1-2 bulan |
| Time to $1K MRR | 12-18 bulan | 6-9 bulan |
| 1-year survival rate | 40-50% | — |
| Conversion free → paid | 2-5% | 5-10% |
| Solo founder burnout rate (12 mo) | 30-40% | — |

### Adjustment dari Inside View

CatatOrder memiliki beberapa faktor yang membuatnya LEBIH BAIK dari median:
- (+) Produk sudah lengkap (kebanyakan micro-SaaS launch MVP minimal)
- (+) Biaya sangat rendah (survival tidak tergantung revenue)
- (+) Masalah nyata yang tervalidasi (200+ jam riset)
- (+) Timing baik (kompetitor gugur)

Dan beberapa faktor LEBIH BURUK:
- (-) Target market sulit dijangkau (UMKM mikro Indonesia, bukan tech-savvy users)
- (-) Revenue per user sangat rendah (Rp15K vs $10-50/mo typical SaaS)
- (-) Solo founder tanpa co-founder distribusi
- (-) Zero existing audience atau network

**Adjusted estimate:**
- Time to first paying customer: 2-4 bulan (sedikit lebih baik dari median karena produk sudah siap)
- Time to Rp1.4M MRR (≈$85): 9-15 bulan
- 1-year survival: 60-70% (lebih baik dari median karena ultra-low cost)

## Game Theory: Analisis Insentif Aktor

### Aktor 1: UMKM F&B (Target Customer)

| Insentif | Kekuatan |
|----------|----------|
| Mengurangi pesanan hilang | **Tinggi** (langsung terasa di pendapatan) |
| Hemat waktu rekap | **Sedang** (1-3 jam/hari) |
| Terlihat profesional (struk digital) | **Sedang** (prestige value) |
| Status quo (buku tulis) | **Tinggi** (kebiasaan kuat, "sudah cukup") |
| Menghindari biaya baru | **Tinggi** (budget ketat) |

**Equilibrium:** UMKM akan coba CatatOrder hanya jika:
- (a) Gratis (✅ 50 pesanan/bulan)
- (b) Direkomendasikan teman tepercaya (belum terbentuk)
- (c) Baru saja mengalami kerugian dari pesanan berantakan (trigger moment)

### Aktor 2: Pelanggan End-User (Customer dari UMKM)

| Insentif | Kekuatan |
|----------|----------|
| Kemudahan pesan via link (vs chat WA manual) | **Sedang** (convenience) |
| QRIS payment langsung | **Sedang** (modern payment) |
| Struk digital sebagai bukti | **Rendah-Sedang** |
| Kebiasaan chat WA langsung | **Tinggi** (status quo) |

**Insight:** End-user tidak aktif mencari CatatOrder — mereka terexpose secara pasif. Link toko harus LEBIH MUDAH dari chat WA untuk menghasilkan adoption.

### Aktor 3: Kompetitor Potensial

| Aktor | Insentif Masuk | Barrier |
|-------|---------------|---------|
| WhatsApp/Meta | Rendah (order management bukan core) | Tidak fokus pada micro-UMKM |
| GoTo/Gojek | Rendah (sudah punya Moka POS) | Moka untuk segment berbeda |
| New startup | Sedang (jika CatatOrder terbukti) | Perlu funding di winter VC |
| Existing UMKM tool (BukuWarung) | Rendah (sudah pivot ke fintech) | Tidak akan kembali |

**Assessment:** Risiko kompetitor dalam 12 bulan = **RENDAH**. Window of opportunity terbuka.
