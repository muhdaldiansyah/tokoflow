## 3. User Pain Points

### The Core Problem

**Manual WA order processing wastes 2-4 hours/day:**

1. **Unformatted messages:** "kak mau pesen dong" — requires 4-8 follow-up messages per order (DOKU Blog)
2. **Missed orders:** Dozens to hundreds of chats/day. 89% of consumers expect reply within 15 minutes. 90% switch to competitors if no fast response (CRM.id, Kirimi.id)
3. **Manual recording errors:** Copying chat → notebook/spreadsheet = error-prone. Rahayu Bakery case: incomplete data, difficulty tracing orders (academic paper)
4. **Mixed personal + business chat:** Easy to miss orders in personal messages (CRM.id)
5. **No customer data retention:** 69% of operators don't record chat history, customers repeat info every time (Kirimi.id)
6. **Time drain:** With OMS, processing drops from "several hours" to <30 minutes, cancellations drop 90% (Dazo)

### Ramadan Peak Pain

- Order volume 2-5x during Ramadan
- Pre-order systems add complexity (advance ordering for iftar, events)
- 20-50+ orders/day = acute chaos
- Missing one catering order = Rp200K-Rp2M+ lost revenue

5. **Stage 5 (rare):** Full OMS (Dazo, Opaper POS) → structured, automated
