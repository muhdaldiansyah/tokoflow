## CRITICAL (1)

### 9.1 Production export route NO auth check
`/api/production/export` tidak panggil `getAuthenticatedClient`. Rely on header forwarding ke internal fetch — fragile.
- **Fix:** Tambah explicit auth check

## HIGH (6)

### 1.4 Receipt number collision risk
Receipt pakai 4-digit random — no uniqueness check, no DB constraint. Collision 1/10000 per second.
- **Fix:** Sequential counter atau UNIQUE constraint + retry

### 2.1 Referral commission race condition
Read-modify-write pattern: read balance → add → write. Two simultaneous payments = lost commission.
- **Fix:** Atomic increment: `referral_balance = referral_balance + commission`

### 3.1 WA bot no rate limit on outgoing messages
Meta API punya rate limit. Bot reply tanpa throttle bisa kena block.
- **Fix:** Per-customer rate limiter (max 10 msg/min)

### 4.1 Order items no server-side price/qty validation
Client bisa submit `price: -1000, qty: 0`. Server terima tanpa validasi.
- **Fix:** Validate per item: name non-empty, price >= 0, qty >= 1

### 4.3 Slug update no format/reserved word validation
`isValidSlug()` dan `isReservedSlug()` exist di utils tapi TIDAK dipakai di API route. Slug bisa diset ke "api", "login", "admin".
- **Fix:** Import dan panggil validasi functions

### 8.1 Storage signed URL no bucket allowlist
Client bisa request signed URL untuk ANY bucket. Service role bypasses policies.
- **Fix:** Allowlist: `['product-images', 'order-images', 'avatars', 'qris']`
